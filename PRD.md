# PRD — 딱 (Ddak)

> **Product Requirements Document**  
> 버전: 1.0 | 작성일: 2026-06-20

---

## 1. 제품 개요

**딱(Ddak)** 은 큰 목표를 2분짜리 마이크로 태스크로 쪼개어 시작을 돕는 감정 반응형 웹 앱입니다.

- **슬로건:** "딱 2분, 딱 이것만, 딱 시작"
- **핵심 철학:** 완성하려 하지 말고, 그냥 시작만 해

---

## 2. 해결하는 문제

| 문제 | 설명 |
|------|------|
| 심리적 시작 장벽 | 두려움·완벽주의·압도감으로 인해 일을 미루는 현상 |
| 과제의 거대함 | 목표가 너무 커서 첫 행동을 정하지 못함 |
| 동기 부족 | 진행 중임을 시각적으로 체감하기 어려움 |

---

## 3. 타겟 사용자

모든 종류의 일을 미루는 사람 — 일상적 할 일부터 대형 프로젝트까지.  
별도 회원가입 없이 즉시 사용 가능한 형태로 진입 장벽을 최소화합니다.

---

## 4. 핵심 기능

### 4.1 AI 태스크 분해
- 사용자가 큰 목표를 자연어로 입력
- 서버의 `GoalCoachAgent`가 OpenAI를 호출해 **5~10개의 2분짜리 마이크로 태스크**로 분해
- 동시에 **단일 감정 태그** (burden / creative / difficult / urgent / routine / new) 분류
- 반환 데이터는 zod 스키마로 검증된 JSON

**에이전트 파이프라인:**
1. `goal_intake` — 입력 목표 정규화
2. `safety_review` — 유해·불법 목표 차단, 고위험(의료·법률·금융) 목표 경고
3. `task_decomposition` — 2분 내 완료 가능한 물리적 액션 5~10개 생성
4. `structured_output` — 스키마 검증 후 JSON 반환

### 4.2 딱 2분 모드 (현재 태스크 카드)
- 화면 중앙에 **첫 번째 태스크 하나만** 크고 명확하게 표시
- "2분이면 돼요" 태그로 심리적 부담 완화
- [시작!] 버튼 클릭 시 파티클 폭발 애니메이션
- 나머지 태스크는 하단에 흐릿하게 표시 → 압도감 제거

### 4.3 감정 반응형 3D 오브젝트 (핵심 차별점)
감정 태그에 따라 배경 3D 오브젝트의 형태·질감·색상·움직임이 달라집니다.

| 감정 태그 | 오브젝트 형태 | 색상 | 움직임 |
|----------|------------|------|--------|
| 😰 burden (부담) | Dodecahedron (돌 질감) | 회색, 카키, 머드 | 느리고 무겁게 흔들림 |
| 🎨 creative (창의) | Blob (물방울형) | 무지개, 파스텔 | 경쾌하게 튀어오름 |
| 🔪 difficult (복잡) | Icosahedron (스파이크) | 레드, 퍼플 | 진동하듯 떨림 |
| 🔥 urgent (긴급) | Spike (불꽃형) | 오렌지, 레드 | 빠르게 진동 |
| 😴 routine (반복) | FlatBlob (납작) | 베이지, 연한 회색 | 거의 안 움직임 |
| 🌱 new (새로 시작) | Seed (씨앗/새싹) | 연두, 민트 | 천천히 자라나는 듯 |

- 모바일 저성능 환경: 3D 렌더링 → **2D CSS 애니메이션 자동 폴백** (progressive enhancement)

### 4.4 진행 및 완료 UX
- 태스크 완료 시 3D 오브젝트가 점점 부드럽고 가볍게 변형
- 완료 후 **기분 이모지 체크** (😊 😤 😴 등) → 다음 태스크 톤 조정
- 전체 태스크 완료 시 오브젝트 폭발 + 축하 메시지
- **명예 시스템:** "완료했어요!" 버튼 하나, 별도 검증 없음

### 4.5 재방문 자동 복귀
- 미완료 태스크가 있으면 마지막 진행 화면으로 자동 복귀
- "계속할까요?" 프롬프트로 이탈 후 재진입 마찰 최소화
- 상태는 **LocalStorage**에 저장 (서버 DB 없음)

## 5. 사용자 흐름

```
홈 화면
└── 입력창: "오늘 뭘 해야 하나요?"
    └── 기본 blob 배경 애니메이션

AI 처리 화면
└── 로딩: 물방울이 작게 쪼개지는 애니메이션
    └── 감정 태그 분류 + 오브젝트 변형 시작

메인: "지금 할 일" 화면
├── 배경: 감정 태그에 맞는 3D 오브젝트
├── 현재 태스크 카드 (중앙, 크게)
│   ├── "2분이면 돼요" 태그
│   └── [시작!] 버튼
└── 나머지 태스크 (하단, 흐릿하게)

완료 화면
└── 오브젝트 부드러워짐 + 다음 단계 자동 등장
    └── 전체 완료 → 오브젝트 폭발 + 축하 메시지
```

**예시:**
```
입력: "세금 신고서 작성하기"
  → 감정 태그: burden → 돌 질감 오브젝트
  → 1단계: "국세청 홈택스 사이트만 열어봐" (2분)
  → 완료! → 오브젝트 살짝 부드러워짐
  → 2단계: "작년 신고서 파일 찾아봐" (2분)
  → ...
  → 전체 완료 → 오브젝트 폭발 🎉
```

## 7. API

### `POST /api/analyze`
목표 텍스트를 받아 태스크 목록과 감정 태그를 반환합니다.

**Request:**
```json
{ "goal": "논문 초안 쓰기" }
```

**Response:**
```json
{
  "tasks": [
    "빈 문서 열고 제목만 써",
    "논문 개요 3줄 적기",
    ...
  ],
  "emotionTag": "burden"
}
```

- Rate Limit: IP당 분당 5회
- API 키는 서버 환경 변수로만 관리, 클라이언트 노출 금지
- Azure OpenAI 우선, 없으면 `OPENAI_API_KEY` 폴백

### `GET /api/health`
배포 상태 확인용 헬스체크 엔드포인트.

## 8. 보안 & 성능

- OpenAI API 키는 서버사이드 전용 (`/api/analyze` Route Handler)
- IP 기반 Rate Limiting (분당 5회, HttpOnly 쿠키 + 인메모리)
- 모바일 저성능 감지 → 3D에서 2D CSS 폴백 자동 전환
- 모델 자격증명은 로그·클라이언트 번들·브라우저 스토리지에 절대 노출 금지
- 안전하지 않은 목표(불법, 자해 등) 자동 차단

## 10. 데모 시나리오

심사 시 다음 3가지 플로우로 시연합니다.

1. **논문 쓰기** — 대형 작업을 첫 문장/파일 열기 액션으로 쪼개는 부담 경감 효과 시연
2. **방 정리하기** — 루틴 생산성, 첫 물리적 이동 액션으로 전환하는 효과 시연
3. **포트폴리오 만들기** — 창의적/새 작업을 프로젝트 폴더·제목·첫 섹션부터 시작하는 효과 시연
