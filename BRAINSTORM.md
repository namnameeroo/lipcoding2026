## 📌 앱 개요

**앱 이름:** **딱 (Ddak)** — "딱 2분", "딱 이것만", "딱 시작"

**컨셉:** 어떤 일이든 시작하기 쉽도록 도와주는 웹 애플리케이션

**핵심 철학:** *"완성하려 하지 말고, 그냥 시작만 해"*

**차별점:** AI 스마트 태스크 분해 + 감정 반응형 3D 오브젝트 — 두 기능이 동등하게 핵심. AI가 감정 태그를 분류하고, 그 태그가 3D 오브젝트를 결정하는 파이프라인이 핵심 루프

---

## 🎯 타겟 & 핵심 가치

- **타겟:** 모든 종류의 일을 미루는 사람 (일상적 할 일 ~ 대형 프로젝트)
- **핵심 문제:** 시작을 못 하는 심리적 장벽 (두려움, 완벽주의, 압도감)
- **해결 방식:** 작업을 자동으로 작게 쪼개서, 첫 2분짜리 액션만 보여줌 + 시각적 감정 표현

---

## 💡 핵심 기능

### 1. 스마트 태스크 분해 (AI)
- 사용자가 큰 목표를 입력하면 AI가 자동으로 **5~10개 마이크로 태스크**로 분해
- 화면에는 **첫 번째 태스크만** 크게 표시
- 나머지는 흐릿하게 숨김 → 압도감 제거

### 2. 딱 2분 모드
- "이것만 하면 돼" → 딱 **2분짜리 액션** 하나만 제시
- 예: "논문 쓰기" → _"지금 빈 문서 열고 제목만 써"_
- **구현:** 별도 시간 검증 없이 프롬프트로 강제 — `"절대 2분 안에 완료 가능한 단 하나의 물리적 행동으로 표현하세요"`

### 3. 🌟 감정 반응형 3D 오브젝트 (핵심 차별점)
- 태스크 입력 시 AI가 **감정 태그 하나**만 자동 분류 (가장 지배적인 감정)
- 태그에 따라 배경 3D 오브젝트의 **형태·질감·색상·움직임**이 달라짐
- 모바일 저성능 환경에서는 **2D CSS 애니메이션으로 폴백** (progressive enhancement)

### 4. 시각적 보상 효과
- 시작 버튼 누르면 **파티클 폭발 애니메이션**
- 태스크 완료 시 오브젝트가 점점 **부드럽고 가볍게 변형**
- 완료 후 **기분 이모지 체크** (😊 😤 😴 등) → 다음 태스크 톤 조정에 활용
- 모든 완료 시 오브젝트 **폭발 후 흩어짐**

### 5. 재방문 UX
- 미완료 태스크가 있으면 마지막 진행 중이던 화면으로 **자동 복귀**
- "계속할까요?" 프롬프트로 이탈 후 재진입 마찰 최소화
- LocalStorage에 상태 저장

## 🎭 감정 태그 & 오브젝트 시스템

> **규칙:** AI가 태스크당 **단일 태그**만 분류 (가장 지배적인 감정 하나)

| 태스크 유형 | 오브젝트 특성 | 색상 | 움직임 |
|------------|------------|------|--------|
| 😰 **부담스러운 일** | 돌 질감, 묵직한 형태 (Dodecahedron) | 회색, 카키, 머드 | 느리고 무겁게 흔들림 |
| 🎨 **창의적인 일** | 부드러운 blob, 물방울형 | 무지개, 파스텔 혼합 | 경쾌하게 튀어오름 |
| 🔪 **까다로운/복잡한 일** | 뾰족한 스파이크 (Icosahedron) | 날카로운 레드/퍼플 | 진동하듯 떨림 |
| 😴 **지루한/반복적인 일** | 납작하고 흐릿한 형태 | 베이지, 연한 회색 | 거의 안 움직임 |
| 🔥 **긴급한 일** | 불꽃/뾰족 + 빠른 형태 | 오렌지, 레드 | 빠르게 진동 |
| 🌱 **새로 시작하는 일** | 씨앗/새싹 형태 | 연두, 민트 | 천천히 자라나는 듯 |

## 🎨 디자인 방향

- **스타일:** 유동적(Fluid) & 유기적(Organic) + 애니메이션
- **배경:** 느리게 움직이는 3D 오브젝트 (감정 반응형)
- **카드:** Glassmorphism (반투명 유리 스타일)
- **전환:** Framer Motion `spring` 애니메이션
- **기본 컬러:** 라벤더 → 민트 → 코랄 그라디언트

## 📱 페이지 & UX 플로우

```
홈 화면
└── 큰 입력창: "오늘 뭘 해야 하나요?"
    └── 배경: 기본 blob 애니메이션

AI 처리 화면
└── 로딩: 물방울이 작게 쪼개지는 애니메이션
    └── 감정 태그 분류 + 오브젝트 변형 시작

🌟 메인: "지금 할 일" 화면
├── 배경: 감정 태그에 맞는 3D 오브젝트 (느리게 움직임)
├── 딱 하나의 카드 (크게, 중앙)
│   ├── "2분이면 돼요" 태그
│   └── [시작!] 버튼 → 파티클 폭발
└── 나머지 태스크 (아래 흐릿하게)

완료 화면
└── 오브젝트가 점점 부드러워짐 + 다음 단계 자동 등장
    └── 모두 완료 → 오브젝트 폭발 + 축하 메시지
```

### UX 플로우 예시
```
입력: "세금 신고서 작성하기"
  → 감정 태그: [부담, 복잡] → 돌 질감 + 스파이크 혼합 오브젝트
  → 1단계: "국세청 홈택스 사이트만 열어봐" (2분)
  → 완료! → 오브젝트가 살짝 부드러워짐
  → 2단계: "작년 신고서 파일 찾아봐" (2분)
  → ...
  → 전체 완료 → 오브젝트 폭발 🎉
```

## 🛠 기술 스택

```
Frontend
├── Next.js 14 (App Router)
├── TypeScript
├── Tailwind CSS (레이아웃, 기본 스타일)
├── Framer Motion (UI 전환 애니메이션)
├── React Three Fiber + Three.js (3D 오브젝트, 데스크탑)
├── @react-three/drei (Three.js 헬퍼)
└── Leva (개발 중 파라미터 실시간 조정)

AI
└── OpenAI API (서버사이드 전용, 클라이언트에 키 노출 금지)
    ├── 태스크 분해 (GPT-4o-mini)          ← 비용 절감
    └── 감정 태그 분류 (burden / creative / difficult / urgent / routine / new)

State Management
└── Zustand — 최소 범위만
    ├── tasks: 분해된 태스크 목록
    ├── emotionTag: 현재 감정 태그
    └── currentIndex: 진행 단계
    (그 외 입력값·로딩 상태 등은 React 로컬 state)

Storage
└── LocalStorage — 미완료 태스크 상태 유지 (재방문 복귀용)
```

## 🔒 API 보안 & 성능

- OpenAI API 키는 **서버사이드 전용** (`/api/analyze` Route만 호출)
- IP당 분당 **5회 rate limit** 적용
- 모바일 저성능 감지 시 3D 렌더링 → **2D CSS 폴백** (progressive enhancement)

## 💡 완료 처리 방식

- **명예 시스템:** "완료했어요!" 버튼 하나, 별도 검증 없음
- 완료 버튼 클릭 시 **기분 이모지 체크** → 다음 태스크 감정 톤 조정에 활용
- 완료 시 3D 오브젝트 점점 부드러워짐, 전체 완료 시 폭발 애니메이션
